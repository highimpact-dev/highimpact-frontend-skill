The Problem

You Know It When You See It

Inter. Blue #2563eb. "Detect Threats Before They Become Breaches." Congratulations — you have the same page as everyone else.

Without the plugin

Claude picks the safe option. Inter because it's readable. Blue because it's trustworthy. Headlines that explain the category instead of owning it. It's not bad — it's invisible. Technically correct, completely forgettable.

With the plugin

The brief hits first. Discovery questions narrow the audience. A conceptual anchor shapes every choice that follows. Exo 2 because surveillance-grid geometry earns credibility. Deep navy because it's earned, not assumed. A headline that puts the reader inside the problem they're trying to avoid.

Process

Ten Steps From Prompt to Designed

The plugin doesn't suggest better fonts. It installs a process that makes better choices unavoidable.

1
Discovery

Read the Room

Detects project context and asks targeted questions — audience, tone, competitive position — before touching a color or typeface.

2
Design Plan

Anchor Before You Build

Produces a conceptual anchor, font selection with rationale, color system, and layout direction as a committed brief, not a suggestion.

3
Style Guide

Proof Before Code

Renders a live style guide in the browser — typography at scale, color relationships, spacing rhythm — so you approve the design system before a single component gets built.

4
Copywriting

A Copywriter Shows Up

A dedicated subagent writes every headline, label, body block, and CTA from the brief — not placeholder text, not "lorem ipsum," not "Your headline here."

5
Staged Build

Four Verified Stages

Structure → typography → layout → atmosphere. Each stage completes and verifies before the next one starts. No half-built components hiding behind working ones.

6
Live Testing

Screenshots, Not Guesses

Takes browser screenshots at mobile, tablet, and desktop breakpoints. Reviews them against the design plan. Catches drift before you do.

7
Design Review

Plan vs. Reality Check

Compares the built output against the original brief line by line. Typography contrast. Color accuracy. Conceptual fidelity. Flags deviations instead of hoping you don't notice.

8
Refinement Loop

Iterate Until It's Right

Stays in the loop — reviewing, adjusting, verifying — until the output matches the plan or you say stop.

9
Project Memory

Learns Your Preferences

Remembers typeface preferences, spacing decisions, tone choices, and anti-patterns from your previous builds so you stop re-explaining yourself.

10
Skill Evolution

Gets Smarter Across Projects

Captures cross-project learnings and auto-generates new anti-rationalization rules from patterns it catches itself trying to repeat.

Font Selector

1,500+ Fonts. The Right One in One Query.

Not a dropdown. A semantic search across 20 mood dimensions — so "corporate but dangerous" finds something that actually looks corporate but dangerous.

Most font tools are dictionaries. This one is a translator. Describe the feeling you're going for — the audience's trust signals, the brand's emotional register, the tension the design needs to hold — and get back fonts that were chosen because they fit, not because they appeared first alphabetically.

Active Artistic Awkward Business Calm Childlike Competent Cute Excited Fancy Futuristic Happy Innovative Loud Playful Rugged Sincere Sophisticated Stiff Vintage
font-selector — node select-fonts.mjs
Example query:
node select-fonts.mjs '{"Futuristic": 80, "Competent": 85, "Innovative": 75}'
Result

Returns Exo 2 (geometric precision, surveillance-grade structure), Ubuntu (technical authority with approachability), and Saira Condensed (high-information-density, military register).

Every font pairing includes rationale — why this combination fits this emotional brief, and which weight/size combinations to avoid.

Enforcement

The Model Will Rationalize. This Stops It.

AI models default to the statistical center of everything they've seen. That center is Inter, blue gradients, and headlines that describe the product category without saying anything about the product. The anti-rationalization tables are the plugin's enforcement mechanism — a list of every shortcut the model will attempt, closed before it can negotiate its way through them.

Every row below is a real argument Claude will make. The reality column is why it doesn't hold.

The Excuse
"Inter is clean and readable."
The Reality

So is every other page. Readable is table stakes. Readable and memorable requires a choice.

The Excuse
"I'll simplify the responsive strategy — same result, less complexity."
The Reality

The design plan said three breakpoints for a reason. Simplified responsive isn't equivalent. It's a narrower page pretending to be a site.

The Excuse
"This animation is too complex for this use case."
The Reality

The signature moments are what make a design feel intentional instead of assembled. If the only complex part gets cut, you kept the scaffolding and lost the building.

The Excuse
"I'll skip the texture — it's a subtle detail."
The Reality

That's the tell. Designed pages have texture. Generated pages don't. Subtle details are exactly how you know which one you're looking at.

The Excuse
"The user just wants working code fast."
The Reality

The user also said they were tired of generic output. A 15-line design plan takes 30 seconds. Skipping it doesn't save time — it saves the model from doing the hard part.

The Excuse
"I'll tighten the copy a bit — just cleaning it up."
The Reality

The copywriter wrote those words for a reason. "Cleaning up" means averaging back toward the statistical center. You're a builder. Ship the copy you were given.

The Excuse
"This color is close enough."
The Reality

#2563eb and #1E3A5F are not the same decision. One is a default. One is a choice. The plan specified a choice.

The Excuse
"I'll use a standard grid — it's more flexible."
The Reality

Flexible means nothing was decided. The plan isn't a suggestion. Build what was designed.

These aren't hypothetical. Every one of these shortcuts has appeared in real plugin runs. The tables exist because "close enough" compounds — five small rationalizations and you're back to a template.

Architecture

One Orchestrator. Specialists That Know Their Job.

The plugin runs as an orchestrator that delegates to purpose-built subagents. Each agent has a narrow brief, its own anti-rationalization rules, and a single job it can't skip its way through.

Orchestrator

Manages the 10-step workflow. Enforces stage gates. Nothing moves forward without verification.

System
Font Selector

Semantic search across 1,500+ Google Fonts using 20 mood dimensions. Returns fonts with rationale.

Sonnet
Copywriter Agent

Writes every piece of text from the brief. Has its own banned-patterns list.

Sonnet
Builder Agent

Four-stage construction with verification gates.

Sonnet
Tester Agent

Browser screenshots at each breakpoint.

Haiku
Reviewer Agent

Compares output against the plan. Flags drift.

Haiku
Gate Agent

Final quality verification before ship.

Memory
Project Memory

Persists preferences across builds.

Memory
Cross-Project Learnings

Auto-evolving global patterns.

Each agent can't skip its job. The orchestrator can't move to the next stage without a passing verification. That's not a guardrail — that's the process.
